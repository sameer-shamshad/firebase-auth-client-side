/*
===============================================

Firebase Auth - UserCredential Object (Overview)

===============================================

When you sign in, sign up, or sign in with providers,
Firebase returns a "UserCredential" object.

It contains THREE main things:

1. user
2. providerId
3. operationType
4. (Optional) additionalUserInfo  <-- appears in OAuth flows (Google, GitHub, etc.)

-----------------------------------------

1) user (Most important!)

-----------------------------------------

This is a "User" object representing the signed-in user.

user = {
  uid: string                        // Unique user ID generated by Firebase
  email: string | null               // User's email if available
  emailVerified: boolean             // Whether the user's email is verified
  displayName: string | null         // User's full name (mostly provider login)
  photoURL: string | null            // Profile photo URL
  phoneNumber: string | null         // Phone number if phone auth
  isAnonymous: boolean               // True if user signed in anonymously

  metadata: {
    creationTime: string             // Account creation time
    lastLoginTime: string            // Last sign-in time
  }

  providerData: [                    // Array of provider-specific info
    {
      uid: string                    // Provider UID
      providerId: string             // "google.com", "github.com", etc.
      email: string | null
      phoneNumber: string | null
      displayName: string | null
      photoURL: string | null
    }
  ]

  stsTokenManager: {                 // Internal token manager
    accessToken: string              // Firebase Auth JWT
    expirationTime: number           // Expiration timestamp (ms)
    refreshToken: string
  }

  tenantId: string | null            // For multi-tenant projects

  // Methods available on user:
  async getIdToken()                 // Returns that user's ID token (JWT)
  async reload()                     // Refreshes user data
  async updateEmail(newEmail)
  async updatePassword(newPassword)
  async updateProfile({displayName, photoURL})
  async delete()                     // Deletes user account
}

-----------------------------------------

providerId

-----------------------------------------
Shows which provider was used for sign-in:

- "password"           (Email & Password)
- "google.com"
- "github.com"
- "facebook.com"
- "phone"
- "anonymous"

-----------------------------------------

operationType

-----------------------------------------
Specifies what auth operation just happened:

- "signIn"     → User logged in
- "signUp"     → User created an account
- "link"       → Provider was linked to existing user
- "reauthenticate"

-----------------------------------------

additionalUserInfo (OAuth only)

-----------------------------------------
This value exists only for provider logins (example: Google).

additionalUserInfo = {
  isNewUser: boolean                // True if this is the user’s first login
  providerId: string                // Provider used
  profile: object                   // Raw provider profile (Google, GitHub, etc.)
  username: string | null           // GitHub/Twitter usernames if available
}

-----------------------------------------

Summary:

-----------------------------------------
userCredential = {
  user,                 // Most commonly used (contains full user object)
  providerId,           // Provider used for login
  operationType,        // What auth action happened
  additionalUserInfo    // (Optional) Provider profile info
}
*/
